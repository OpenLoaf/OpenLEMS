# Contributing

感谢你关注这个项目。

## 提交前请先确认

- 不要提交任何真实设备数据、日志、数据库、证书、密钥或账号配置。
- 保持变更聚焦，避免把无关重构混入同一个提交。
- 新增或修改功能后，优先补充最小必要的测试或验证说明。

## 本地检查

在提交 PR 前，建议至少执行：

```bash
./script/build-workspace.sh
```

如需补充或排查测试，可执行：

```bash
./script/test-workspace.sh
```

## Pull Request

- 在 PR 描述中写清楚变更目的、主要修改点和验证方式。
- 如果涉及接口、配置或数据结构变化，请同时更新相关文档。
- 如果涉及兼容性风险，请在描述中明确说明影响范围。
