# Cursor Rules 优化 CICD 流程

## 流程概述

本 CICD 流程用于自动化管理和优化 Cursor 项目规则（rules），确保规则集的完整性、一致性和高效性。

## CICD 流程定义

### 阶段 1: 上下文分析与规则识别

#### 1.1 用户上下文总结

- 检查当前打开的文件和目录
- 识别用户正在处理的模块或功能
- 分析最近的代码变更和提交记录
- 确定用户当前的技术栈和开发阶段

#### 1.2 规则匹配与提取

- 扫描 `.cursor/rules/` 目录下的所有 `.mdc` 文件
- 解析规则文件的元数据（`alwaysApply`, `description`）
- 提取规则的核心内容和适用范围
- 将用户上下文与现有规则进行匹配

### 阶段 2: 内容变更分析与规则更新

#### 2.1 变更内容分析

- 分析用户提交的代码变更
- 分析当前未提交的代码变更
- 识别新增、修改、删除的代码模式
- 检测新的命名约定、架构模式或技术栈变更
- 检查 README、文档文件的更新

#### 2.2 规则更新决策

- **新规则识别**：判断是否需要新增规则
  - 新的技术栈或框架引入
  - 新的架构模式或设计模式
  - 新的命名约定或代码规范
  - 新的开发流程或最佳实践
  - **重要限制**：除非用户明确指定，否则不要创建新的规则文件

- **现有规则更新**：判断是否需要更新现有规则
  - 规则内容过时或不再适用
  - 规则描述不够清晰或完整
  - 规则适用范围需要调整

### 阶段 3: 规则优化与合并

#### 3.1 重复规则检测

- 扫描所有规则文件，识别重复或相似的规则内容
- 检测规则间的重叠和冲突
- 分析规则表达方式的差异

#### 3.2 规则合并策略

- **合并原则**：
  - 保留最完整的版本
  - 保持语义完整性
  - 维护向后兼容性
- **合并操作**：内容合并、规则拆分、规则重构
