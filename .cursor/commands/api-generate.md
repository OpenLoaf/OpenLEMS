# API 接口生成命令集合

## 命令概述

本命令集合用于指导 AI 自动生成符合项目规范的 API 接口代码，包括接口定义、控制器实现、实体模型等完整代码。

## 命令类型

### 1. 快速生成模式 (`@quick-api`)

**适用场景**：需要快速创建标准化的 API 接口

**特点**：简洁明了，执行步骤清晰

### 2. 标准生成模式 (`@api-generator`)

**适用场景**：需要完整了解接口生成流程

**特点**：包含详细的规范说明和检查清单

### 3. 完整生成模式 (`@generate-api-interface`)

**适用场景**：需要深入了解接口生成的每个细节

**特点**：包含完整的执行流程、规范说明和最佳实践

## 统一执行流程

### 1. 需求分析阶段

-**分析用户输入**：仔细理解用户描述的接口需求

-**确定模块名称**：根据功能确定合适的模块名称（如 user、device、alarm 等）

-**识别接口类型**：判断是查询类接口还是操作类接口

-**确定接口动作**：识别具体的操作（如 get、create、update、delete、list 等）

-**分析数据结构**：理解请求和响应的数据结构需求

### 2. 规范检查阶段

-**参考规范文档**：严格按照 `@application-api-development.mdc` 规则执行

-**检查命名规范**：确保模块名、接口名、结构体名符合项目规范

-**验证文件结构**：确认目录结构和文件命名符合标准

-**检查数据类型**：确保时间字段使用 `*time.Time` 类型

### 3. 代码生成阶段

#### 3.1 创建 API 接口定义

-**文件位置**：`api/[module]/v1/[module]_[action].go`

-**请求结构体**：`[Action][Module]Req`

-**响应结构体**：`[Action][Module]Res`

-**操作类接口**：使用空响应结构体 `type UpdateSettingRes struct {}`

-**查询类接口**：返回具体数据结构

#### 3.2 更新模块接口定义

-**文件位置**：`api/[module]/[module].go`

-**添加新接口方法**：在模块接口中添加对应的方法定义

#### 3.3 创建实体模型

-**文件位置**：`internal/model/entity/[module].go`

-**实现 UnmarshalValue 方法**：用于数据转换

-**时间字段处理**：使用 `*time.Time` 类型

#### 3.4 实现控制器

-**文件位置**：`internal/controller/[module]/[module]_v1_[action].go`

-**构造函数**：创建 `[module]_new.go` 文件

-**操作类接口**：成功时返回空结构体，失败时通过 err 返回

-**查询类接口**：返回具体数据

#### 3.5 注册 Web 服务

-**文件位置**：`internal/cmd/web.go`

-**导入控制器包**：添加控制器包导入

-**绑定控制器**：使用 `group.Bind([module].NewV1())`

### 4. 验证阶段

-**检查文件结构**：确认所有文件都已正确创建

-**验证代码规范**：检查命名、结构体设计、错误处理等

-**确认响应结构体**：操作类接口使用空响应，查询类接口返回数据

-**检查时间字段**：确保使用 `*time.Time` 类型

-**验证错误处理**：使用 `gerror` 进行统一错误处理

### 5. 文档生成阶段

-**创建接口文档**：在接口文件夹下创建 `[module]_[action].md` 文件

-**包含内容**：

- 接口描述和用途
- 请求参数说明
- 响应数据结构
- 错误码说明
- 使用示例
- 注意事项

## 核心规范

### 响应结构体设计

1.**操作类接口**（创建、更新、删除）：

- 使用空响应结构体
- 成功时返回 `&ResponseStruct{}`
- 失败时通过 `err` 返回

2.**查询类接口**：

- 返回具体的数据结构
- 时间字段使用 `*time.Time` 类型

### 时间字段处理

- 统一使用 `*time.Time` 类型
- 支持 `nil` 值表示未设置状态
- 从 `gtime.Time` 转换为 `*time.Time`：`&model.CreatedAt.Time`

### 错误处理规范

-**参数验证错误**：`errors.New("参数不能为空")`

-**业务逻辑错误**：`gerror.NewCode(gcode.CodeNotFound, err, "资源不存在")`

-**内部错误**：`gerror.WrapCode(gcode.CodeInternalError, err, "操作失败")`

-**日志记录**：所有错误都要有详细日志记录

### 命名约定

-**文件命名**：`[module]_v1_[action].go`

-**结构体命名**：`[Action][Module]Req/Res`

-**方法命名**：`[Action][Module]`

-**时间字段**：`CreatedAt`, `UpdatedAt`, `DeletedAt`
