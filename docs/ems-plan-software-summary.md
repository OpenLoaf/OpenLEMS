---

## 一、核心摘要（30秒快速了解）

* **1.1 产品名称**： 微电网开发平台 - 分布式能源管理系统
* **1.2 一句话定位**：
  > **一款专为分布式能源场景设计的，通过插件化架构与多协议支持，解决能源设备管理复杂性和数据孤岛问题的轻量级能量管理系统。**
  >
* **1.3 核心价值**：帮助能源管理团队实现设备统一管理、提升运维效率，并降低系统集成成本的风险。
* **1.4 目标市场/领域**： 智能制造、新能源、工业自动化、智慧能源

---

## 二、问题与解决方案

* **2.1 我们面临的挑战**：
  * **痛点一**： 分布式能源设备种类繁多，协议不统一，设备管理复杂，集成成本高。
  * **痛点二**： 数据采集依赖手动配置，设备状态监控滞后，故障响应不及时。
  * **痛点三**： 传统能源管理系统功能臃肿，学习成本高，部署复杂，维护困难。
* **2.2 我们的解决方案**：
  `EMS Plan` 通过构建一个 `插件化的轻量级能量管理平台` ，首创性地将 `多协议设备驱动` 与 `实时数据推送` 相结合，实现了 `设备管理的标准化、数据采集的自动化、运维决策的智能化`。

## 四、亮点与竞争优势

* **4.1 技术亮点**：
  * **AI智能驱动生成**： 通过AI自动分析用户上传的协议文件，自动生成驱动代码并打包，用户无需手写代码，一杯咖啡时间完成设备从0接入。
  * **插件化架构**： 采用微服务架构和插件化设计，驱动、协议、存储、推送、策略均为可插拔模块，便于扩展与替换，支持热插拔和动态配置。
  * **设备驱动高度定制**： 用户可自定义查询点位、自定义服务、自定义设备配置，适配99%场景，完美支持储能领域的交流母线和直流母线方案。
  * **多协议统一**： 首创性地将Modbus、CANBus、IEC 101/102/104、GPIO等多种工业协议统一管理，通过通用字节解析器支持复杂数据格式，兼容性强。
  * **CGO高性能集成**： 集成C++数值计算库，提供MPC Kalman滤波预测等高性能算法，支持跨平台编译和部署。
  * **全平台支持**： 支持x86、ARM64、ARM32等多种硬件架构，兼容低版本操作系统，适用于从服务器到嵌入式设备的全场景部署。
  * **双模式架构**： 一套系统支持微电网EMS与储能柜EMS模式，通过物模型配置灵活切换。储能联机模式支持主机-从机调度、边缘网关调度、虚拟设备独立运行；微电网模式支持风、光、储、充、荷设备任意接入。
  * **智能运维体系**： 多重环境感知，完整的运维功能支持，支持OTA升级、断点续传、断网自动补发，手机端+Web端多平台配置。
  * **AI云边协同**： AI云策略+本地控制无缝同步，物模型快速适配需求，支持算法与功能持续优化，实现智能决策和自动控制。
  * **内置算法引擎**： 内置微电网和储能算法，用户只需通过AI生成目标设备驱动，即可实现完整的微电网场景，无需复杂算法开发。
* **4.2 用户体验亮点**：
  * **极致易用**： 界面设计遵循工业标准，提供丰富的设备模板和配置向导，新用户无需培训即可快速上手。
  * **高度灵活**： 支持强大的自定义配置，从设备参数到策略规则，均可通过Web界面灵活配置，适配不同场景需求。
* **4.3 市场与生态亮点**：
  * **开放集成**： 提供完整的RESTful API和标准协议接口，支持与SCADA、MES、ERP等主流系统深度集成。
  * **卓越性价比**： 以开源架构提供企业级功能，部署成本降低70%，维护成本降低50%，投资回报周期缩短至3个月。

## 五、目标用户与应用场景

* **5.1 目标用户画像**：
  * **核心用户**： 能源管理工程师、设备运维人员、系统集成商、工业自动化工程师
  * **决策者**： 能源部门总监、CTO、项目负责人、系统架构师
* **5.2 典型应用场景**：
  * **场景一：分布式储能站管理**
    * **背景**： 一个分布式储能站包含多个PCS、BMS、电表等设备，需要统一监控和管理。
    * **流程**： 使用 **插件化设备驱动** 快速接入各类设备 -> 通过 **实时数据采集与监控** 监控设备状态和运行数据 -> 利用 **策略引擎与自动化** 实现智能充放电控制。
  * **场景二：微电网系统集成**
    * **背景**： 微电网系统包含光伏、储能、负载等多种设备，需要统一调度和优化运行。
    * **流程**： 通过 **多存储方案支持** 存储海量运行数据 -> 使用 **数据推送与集成** 与上级调度系统对接 -> 在 **Web管理界面** 中实现可视化监控和远程控制。

## 六、技术信息

* **技术栈**： 后端 Go + GoFrame v2 | 数据库 SQLite + InfluxDB | 前端 Web + WebSocket | 协议 Modbus + CANBus + IEC 101/102/104 + GPIO | 扩展 C++ + CGO
* **部署方式**： 私有化部署、Docker容器化、多架构支持(x86/ARM64/ARM32)
* **安全与合规**： 支持设备认证、数据加密传输、操作日志记录、权限管理
