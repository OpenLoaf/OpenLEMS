# Project Context

## Purpose

EMS Plan 是一个面向分布式能源场景的轻量级能量管理系统（Energy Management System, EMS）。项目的核心目标是：

- **设备统一管理**：提供统一的接口管理 PCS（变流器/逆变器）、BMS（电池管理系统）、电表、消防等各类能源设备
- **插件化架构**：通过可插拔的驱动、协议、存储、推送、策略模块，实现系统的高度灵活性和可扩展性
- **实时数据处理**：完成设备数据的实时采集、状态监控、策略控制与数据持久化
- **多协议支持**：内置 Modbus(TCP/RTU)、CANBus、GPIO 等工业通信协议，支持各类设备接入
- **Web 管理界面**：提供直观的 Web 界面进行设备管理、监控和控制操作
- **策略引擎**：支持基础储能策略和微电网策略，实现智能能源管理

系统采用微服务架构和插件化设计，可快速对接不同厂商的设备，适用于储能电站、微电网、分布式光伏等多种能源管理场景。

## Project Conventions

### Code Style

#### 命名规范
- **接口 (Interface)**: `I` 前缀 - 如 `IDevice`, `IDriver`, `IStorage`
- **结构体 (Struct)**: `S` 前缀 - 如 `SDeviceConfig`, `SDriverInfo`
- **枚举 (Enum)**: `E` 前缀 - 如 `EDeviceType`, `EProtocolType`, `EAlarmLevel`
- **常量 (Constant)**: `C` 前缀 - 如 `CDeviceTypeAmmeter`

#### 文件命名规范 (common/ 目录)
- `c_*_i.go`: 接口定义 (Interface)
- `c_*_s.go`: 结构体定义 (Struct)
- `c_*_e.go`: 枚举定义 (Enum)
- `c_*_f.go`: 函数定义 (Function)
- `c_*_c.go`: 常量定义 (Constant)
- `*_test.go`: 测试文件

#### 代码注解规范
- 所有公共方法、字段、接口都必须有中文注解
- 注解应说明功能、参数、返回值和可能的错误
- 使用 Go 标准注释格式，支持 godoc 生成

#### 包命名规范
- 包名使用小写字母和下划线
- 项目级包以项目首字母小写开头
- 避免与标准库包名冲突

### Architecture Patterns

#### 分层架构
- **应用层 (application/)**: Web 服务、API 接口、命令行工具
- **服务层 (services/)**: 核心业务逻辑编排，如 s_db, s_driver, s_storage
- **插件层 (plugins/)**: 可插拔功能模块 (驱动、协议、存储、策略)
- **通用层 (common/)**: 共享接口、类型定义、基础功能

#### 插件化架构
- **驱动插件 (plug_drivers)**: 设备驱动实现，支持动态加载
- **协议插件 (plug_protocols)**: 通信协议抽象和实现
- **存储插件 (plug_storages)**: 数据存储方案实现
- **策略插件 (plug_policy)**: 能源管理策略实现
- **推送插件 (plug_push)**: 数据推送和上报功能

#### 统一点位系统
- 抽象的设备数据模型，支持不同设备类型的数据统一表示
- 构造函数模式，支持数据映射和转换
- 配置驱动的点位定义，灵活适配不同设备

#### CGO 集成模式
- 性能关键代码使用 C++ 实现
- CGO 提供 Go 与 C++ 的桥接
- 统一的错误处理和资源管理

## Domain Context

### 能源管理行业背景
EMS Plan 专注于分布式能源管理系统，服务于以下应用场景：

#### 主要应用场景
- **储能电站**: 大容量电池储能系统的监控和管理
- **微电网**: 分布式发电、储能和负载的协调控制
- **分布式光伏**: 光伏发电系统的数据采集和性能监控
- **工商业储能**: 工厂、商场等场所的储能系统管理
- **户用储能**: 家庭储能系统的智能化管理

#### 设备类型和厂商
- **PCS (变流器/逆变器)**: 亿兰科 MAC/MDC、星星充电 100E 等
- **BMS (电池管理系统)**: 派能科技 US108、协能等
- **电表**: 安科瑞 10R 等智能电表
- **EMS**: 高特、协能、派能科技 CheckWatt 等能源管理系统
- **消防设备**: 消防控制系统和安全监测设备

### 通信协议标准
系统支持多种工业标准通信协议：

#### Modbus 协议
- **Modbus TCP**: 基于 TCP/IP 的以太网通信
- **Modbus RTU**: 基于 RS-485 的串行通信
- **应用场景**: 电表、PCS、BMS 等设备的标准通信协议

#### CANBus 协议
- **CAN 2.0B**: 标准和扩展帧格式支持
- **UDP 透传**: 通过 UDP 进行 CAN 数据透传
- **应用场景**: 车载设备、工业控制系统

#### GPIO 接口
- **Linux Sysfs**: 通过系统文件接口访问 GPIO
- **数字输入/输出**: 开关量信号采集和控制
- **应用场景**: 设备状态指示、简单控制信号

### 实时数据处理要求

#### 数据采集频率
- **遥测数据**: 1-10 秒采集间隔，根据设备类型可配置
- **状态数据**: 变化触发上报，最小间隔 100ms
- **控制指令**: 实时响应，要求 < 100ms 延迟
- **告警数据**: 立即上报，无延迟

#### 数据精度和完整性
- **模拟量**: 16位精度，支持浮点数和整数
- **开关量**: 布尔值，支持状态变化时间戳
- **字符串**: UTF-8 编码，支持中英文混合
- **数据校验**: CRC 校验、超时检测、重传机制

### 策略引擎和控制逻辑

#### 储能策略
- **充放电控制**: 基于电价、负荷需求的智能调度
- **SOC 管理**: 电池状态管理，防止过充过放
- **功率平滑**: 削峰填谷、功率波动抑制
- **并网/离网**: 支持并网运行和离网运行模式

#### 微电网策略
- **功率平衡**: 发电、储能、负荷的实时平衡
- **电压频率调节**: 通过储能系统稳定电网
- **经济调度**: 基于电价和发电成本的优化调度
- **黑启动**: 支持微电网的黑启动功能

### 行业标准和规范
- **GB/T 34931**: 电化学储能系统储能变流器技术规范
- **GB/T 34131**: 电化学储能电站用电池管理系统技术规范
- **IEC 61850**: 变电站通信网络和系统标准
- **Modbus Application Protocol**: 工业通信协议标准

## Important Constraints

### 开发约束

#### 构建要求
- **开发环境构建**: 必须使用 `-tags=dev` 参数进行开发模式构建
- **CGO 支持**: Application 中必须启用 `CGO_ENABLED=1`
- **Go 版本**: 要求 Go 1.25+ 版本
- **C++ 库编译**: 必须先编译 `@cpp/hexlib` 库才能构建主程序

#### 代码质量约束
- **注解要求**: 所有公共方法、字段、接口都必须有中文注解
- **命名规范**: 严格遵循 I/S/E/C 前缀命名规范
- **错误处理**: 统一使用 `github.com/pkg/errors` 包
- **日志系统**: 使用 `c_log` 包进行统一日志管理

### 部署约束

#### 多架构支持
- **目标平台**: AMD64, ARM64, ARM32 (树莓派等嵌入式设备)
- **Docker 构建**: 必须使用多阶段构建优化镜像大小
- **库依赖**: 固化库路径，确保运行时库文件可用
- **环境变量**: 必须设置 `LD_LIBRARY_PATH` 和 `PATH`

#### 系统资源约束
- **内存使用**: 基础运行内存 < 256MB，建议 512MB+
- **存储空间**: 基础安装 < 100MB，数据空间根据需求配置
- **CPU 要求**: 支持 1 核心运行，建议 2+ 核心
- **网络端口**: Web 服务端口 80 (可配置)，支持端口映射

#### 运行时依赖
- **Linux 内核**: 要求 4.0+ 版本 (GPIO 和其他系统调用支持)

- **权限要求**: 需要访问 GPIO、串口等硬件接口的权限
